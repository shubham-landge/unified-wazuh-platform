# SkillOpt вҖ” Self-Learning Loop Design

> **Status**: Design document; `prompt_refiner.py` is not yet implemented.  
> **Updated**: 2026-06-17  
> **Scope**: How the platform turns analyst feedback into better prompts, models, and autonomy decisions without engineering intervention.

---

## 1. Why SkillOpt matters

The compounding asset of an autonomous SOC is not the LLM вҖ” it is the **decision history**. Every analyst correction, every confirmed verdict, every approved response is a labeled training signal. SkillOpt is the closed loop that:

1. Records agent experiences in `skill_memory`.
2. Surfaces similar past experiences as few-shot context.
3. Detects systematic errors from feedback.
4. Proposes and validates prompt/template improvements.
5. Promotes the best prompts to production with governance.

Without this loop, accuracy plateaus. With it, the platform gets better the longer it runs and the more tenants it serves вҖ” a network effect no single Wazuh deployment can match.

## 2. Data model

### 2.1 Skill memory sources

| Source | Schema | Written by | Used by |
|--------|--------|------------|---------|
| `AgentTask` | `input_data`, `output_data`, `status`, `error` | `agent_worker` on completion | `shared/rag/few_shot.py` |
| `UserFeedback` | `rating`, `correction_text`, `corrected_*` | Dashboard thumbs up/down | `prompt_refiner`, accuracy scoring |
| `AiTriageResult` | `model_name`, `verdict`, `confidence` | `triage_worker` | Model accuracy baseline |
| `knowledge_chunks` (skill_memory:*) | embedded task text | `skill_memory.add_experience` | RAG semantic search |

### 2.2 Feedback taxonomy

A feedback event is a label on a triage or agent decision:

- `rating`: 1вҖ“5 (1 = completely wrong, 5 = perfect)
- `category_correct`: bool
- `severity_correct`: bool
- `correction_text`: free-form analyst explanation
- `corrected_category`, `corrected_severity`, `corrected_confidence`: ground truth

A correction is a **strong signal** that the prompt or model failed for a specific alert class. A run of corrections on the same alert pattern is a **systematic error** and should trigger a prompt refinement.

### 3.1 Step-by-step

1. **Record** (`skill_memory.add_experience`)
   - Every completed `AgentTask` is serialized and stored as a `knowledge_chunk` with source prefix `skill_memory:<agent_type>:<task_id>`.
   - `shared/rag/few_shot.py` retrieves the most recent top-K tasks by agent type.

2. **Retrieve** (`shared/rag/few_shot.retrieve`)
   - Handlers call `await _few_shot("policy_guard", input_data)` to include similar past decisions in the prompt.
   - Future: also call `skill_memory.retrieve_similar()` for semantic matches.

3. **Detect systematic error** (`prompt_refiner.detect_patterns`)
   - Nightly job scans `UserFeedback` where `rating <= 2` or `correction_text` is present.
   - Cluster corrections by `(agent_type, alert_category, mitre_technique, model_name)`.
   - A cluster with вүҘ `SKILLOPT_MIN_CORRECTIONS` (default 5) becomes a candidate for refinement.

4. **Generate prompt draft**
   - For each candidate cluster, build a meta-prompt containing:
     - The current system prompt for the agent
     - Examples of the failed inputs and the corrected outputs
     - The instruction: "Propose a minimal revision to the system prompt that fixes these errors."
   - The LLM returns a revised prompt and a short rationale.

5. **Validate draft**
   - Run the revised prompt against a hold-back set of recent failed cases.
   - Compare predicted verdict/severity/category against the corrected ground truth.
   - Accept only if accuracy improves by вүҘ `SKILLOPT_MIN_IMPROVEMENT` (default 5%).

6. **Shadow / A/B test**
   - Run the new prompt on a percentage of real alerts in shadow mode (record output but do not act on it).
   - Compare shadow accuracy to production over `SKILLOPT_SHADOW_DAYS` (default 7).

7. **Promote**
   - If shadow beats production, create an `ApprovalRequest` of type `prompt_promotion`.
   - A human or a policy-guarded auto-promoter merges the draft into `prompts/<agent>_skill.md` or updates the prompt template in code.
   - Old prompt is archived in `prompts/archive/`.

## 4. Proposed implementation

### 4.1 New file: `services/worker/app/prompt_refiner.py`

Responsibilities:
- `analyze_feedback(session, since: datetime)` вҶ’ list of correction clusters
- `generate_prompt_revision(agent_type, cluster)` вҶ’ draft prompt + rationale
- `validate_revision(agent_type, draft_prompt, test_cases)` вҶ’ accuracy delta
- `promote_revision(agent_type, draft_prompt)` вҶ’ ApprovalRequest or direct update
- `run()` вҶ’ scheduled entry point

Configuration (add to `shared/config.py`):

```python
skillopt_enabled: bool = True
skillopt_min_corrections: int = 5
skillopt_min_improvement: float = 0.05
skillopt_shadow_ratio: float = 0.10
skillopt_shadow_days: int = 7
skillopt_prompt_archive_path: str = "/app/prompts/archive"
```

### 4.2 New scheduled job

Add to `services/worker/app/main.py`:

```python
if settings.skillopt_enabled:
    from app.prompt_refiner import PromptRefiner
    tasks.append(asyncio.create_task(PromptRefiner().run()))
```

Run interval: nightly (`croniter` or simple `asyncio.sleep(86400)`).

### 4.3 Prompt storage

Current prompts live under `prompts/`. Proposed layout:

```
prompts/
  triage.txt              # base prompt
  triage_skill.md         # current learned prompt override
  policy_guard.txt
  policy_guard_skill.md
  archive/
    triage_skill_20260610.md
```

`llm_provider.analyze()` checks for a `_skill.md` file and appends/appends it to the system prompt if present.

### 4.4 Per-agent accuracy scoring

Extend `FeedbackWorker` to compute and store:

```python
agent_accuracy[agent_type].append(rating)
category_accuracy[f"{model}:{category}"].append(category_correct)
severity_accuracy[f"{model}:{severity}"].append(severity_correct)
```

Persist to a new `agent_accuracy` table or append to `ModelRun` rows. This feeds the autonomy ladder UI: an agent with 95% accuracy on `response_planner` can be auto-approved; one with 80% stays in `approval` mode.

- `services/worker/app/prompt_refiner.py`

- `_skill.md` prompt override loader

---

## 6. Governance and safety

- **No automatic prompt promotion without approval** in early deployments. Use `ApprovalRequest` with type `prompt_promotion`.
- **Archive everything** вҖ” old prompts, failed drafts, shadow results.
- **Tenant isolation** вҖ” skill memory can optionally be filtered by `tenant_id`; cross-tenant learning is opt-in and auditable.
- **Prompt-injection guard** вҖ” any learned prompt override must pass `sanitize_llm_input()` before being persisted.

## 7. Success metrics

| Metric | Target | How measured |
|--------|--------|--------------|
| Triage accuracy (thumbs up / total feedback) | +10% after 90 days | `UserFeedback.rating` |
| Analyst corrections per 100 alerts | -30% after 90 days | `UserFeedback` rate |
| Autonomy level advancement | 1 agent moves from `approval` to `full` | Policy-guard audit log |
| Prompt revisions promoted | 1-3 per month | `prompts/archive/` + `ApprovalRequest` |
