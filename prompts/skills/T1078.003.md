# Detection Logic

Monitor for Remote Desktop Protocol (RDP) logons — Windows Event 4624 with LogonType 10. Distinguish between normal administrative access and suspicious lateral movement:

Look for:
- RDP logons from non-IT workstations to servers
- Multiple RDP logons within a short window from the same source
- RDP logons outside business hours (unless shift-based operations)
- RDP logons using non-standard accounts (non-admin, service accounts)
- Source IP geolocation anomalies or known VPN/proxy IPs

# Investigation Steps
1. Verify the source hostname and user — was this an authorized remote session?
2. Check if the same user was also active at their physical workstation at the time
3. Review preceding events for credential access (T1003) or brute-force (T1110)
4. Check for subsequent lateral movement (new RDP sessions, PsExec, WMI) from the target
5. Review session duration — very short sessions may indicate automated tooling

# Response
- If unauthorized: terminate the session, disable the compromised account
- Enable NLA (Network Level Authentication) if not already on
- Review RDP access groups — restrict to required personnel only
- Flag the source IP for increased monitoring across all hosts
