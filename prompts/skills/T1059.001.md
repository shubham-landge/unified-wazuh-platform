# Detection Logic

Monitor for suspicious PowerShell command-line patterns. PowerShell is a heavily-abused execution engine for fileless malware, C2, and lateral movement.

Look for:
- Encoded commands: `-EncodedCommand`, `-enc`, `-e` with base64 payloads
- Download cradles: `IEX (New-Object Net.WebClient).DownloadString`
- Invoke-Expression / IEX patterns
- Execution policy bypass: `-ExecutionPolicy Bypass`, `-EP Bypass`  
- Hidden windows: `-WindowStyle Hidden`, `-W Hidden`
- Non-interactive mode: `-NonInteractive`, `-NonI`
- Obfuscation: excessive backticks, string concatenation, character-level encoding
- Reflection-based loading: `[System.Reflection.Assembly]::Load`

# Investigation Steps
1. Decode any Base64 command lines (first layer at minimum)
2. Check the parent process — was it Office, browser, script, or suspicious binary?
3. Extract any URLs, IPs, or file paths from the command
4. Search DNS/proxy logs for any extracted domains/IPs
5. Check for child processes spawned by PowerShell (WMI, scheduled tasks, service creation)
6. Review PowerShell transcript logs if available

# Response
- If download cradle or reflection detected: isolate the host immediately
- Block extracted IOCs (URLs, IPs, hashes) at firewall/proxy
- Search for the same command pattern across all hosts (retro-hunting)
- Check for successful persistence mechanisms installed after the PowerShell execution
