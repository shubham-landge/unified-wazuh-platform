# Detection Logic

Monitor for newly installed Windows services whose binary path matches known legitimate Windows executables but resides in an unusual directory, or whose service name impersonates a legitimate service with slight misspellings.

Look for:
- Service binary paths outside System32/SysWOW64 for well-known system binaries
- Service names differing by one character from legitimate services (e.g., `svch0st` vs `svchost`)
- Service DisplayName matching known services but binary path not matching
- Services installed in user-writable directories (Temp, AppData, Downloads)

# Investigation Steps
1. Identify the binary path and hash the executable
2. Check file creation timestamp — was it created recently?
3. Compare against known-good service baseline for the host
4. Check parent process that created the service (was it a script interpreter?)
5. Search VirusTotal for the binary hash if the file can be collected

# Response
- If masquerading confirmed: stop the service, set to disabled
- Delete the service and quarantine the binary
- Check for persistence mechanisms (Run keys, scheduled tasks, WMI subscriptions) installed around the same time
- Review outbound network connections from the host for C2 indicators
