# Detection Logic

Monitor for modifications to Windows Run keys and the Startup folder. These are the most common persistence mechanisms for malware, RATs, and backdoors.

Registry paths to monitor:
- `HKLM\Software\Microsoft\Windows\CurrentVersion\Run`
- `HKLM\Software\Microsoft\Windows\CurrentVersion\RunOnce`
- `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
- `HKCU\Software\Microsoft\Windows\CurrentVersion\RunOnce`
- `HKLM\Software\Wow6432Node\Microsoft\Windows\CurrentVersion\Run`

Startup folder paths:
- `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`
- `%ProgramData%\Microsoft\Windows\Start Menu\Programs\Startup`

Look for:
- New Run key values pointing to binaries in Temp, AppData, Downloads, or other user-writable directories
- Binary names masquerading as Windows components (e.g., `svchost.exe` in AppData)
- Script interpreters (powershell.exe, wscript.exe, cscript.exe, mshta.exe) in Run keys
- VBS, JS, HTA, or PS1 files in the Startup folder
- Values with `-WindowStyle Hidden` or `-ExecutionPolicy Bypass` arguments
- Rundll32 or regsvr32 calls loading DLLs from unusual paths

# Investigation Steps
1. Inspect the binary path and hash
2. Check file creation/modification timestamps — correlate with known incident timeline
3. Check the binary on VirusTotal
4. Review scheduled tasks created around the same time (T1053)
5. Check WMI event subscriptions (T1546.003)
6. Identify the user account that created the persistence

# Response
- Delete the suspicious Run key entry
- Quarantine the referenced binary/file
- Check for additional persistence mechanisms (multiple are common)
- Reset affected user credentials if the persistence ran under a user context
