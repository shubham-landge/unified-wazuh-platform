# Detection Logic

Monitor for processes accessing lsass.exe with PROCESS_VM_READ or PROCESS_QUERY_INFORMATION access rights. Common tools include procdump, Task Manager manual dump, comsvcs.dll via rundll32, and mimikatz.

Look for:
- Process access events where TargetImage is lsass.exe and GrantAccess includes 0x1FFFFF or 0x1010
- Command line patterns: `rundll32.exe C:\Windows\System32\comsvcs.dll MiniDump`
- Suspicious parent processes for lsass.exe access (cmd.exe → powershell.exe, wmiexec)

# Investigation Steps
1. Identify the source process and its parent chain
2. Check for lateral movement indicators (PsExec, WMI, WinRM) in the same time window
3. Collect memory dump and process creation logs (Sysmon Event 1)
4. Verify whether the access was from a known security tool (e.g., authorized EDR, defender)
5. Check for subsequent authentication events (Windows Event 4624) with the same source IP

# Response
- If unauthorized: isolate the host immediately
- Collect the offending process binary and submitted to sandbox
- Search for credential use in subsequent logon events across the domain
- Reset credentials for any account active on the affected host
